# Архитектура проекта

Этот раздел описывает архитектуру проекта "Вкатился" и принципы его построения.

## Обзор

Проект использует **Clean Architecture** (Чистую архитектуру) с четким разделением на слои и соблюдением принципов инверсии зависимостей.

## Основные принципы

1. **Разделение ответственности** — каждый слой имеет свою четко определенную роль
2. **Инверсия зависимостей** — зависимости направлены внутрь, к Domain слою
3. **Независимость от фреймворков** — бизнес-логика не зависит от внешних библиотек
4. **Тестируемость** — легкость написания unit-тестов благодаря интерфейсам

## Слои архитектуры

```
┌─────────────────────────────────────┐
│      Presentation Layer             │  ← FastAPI роутеры, DTO
├─────────────────────────────────────┤
│      Application Layer              │  ← Сервисы, фабрики
├─────────────────────────────────────┤
│      Domain Layer                   │  ← Сущности, Use Cases, Интерфейсы
├─────────────────────────────────────┤
│      Infrastructure Layer           │  ← Репозитории, Клиенты, Агенты
└─────────────────────────────────────┘
```

## Технологический стек

- **Backend:** Python 3.11, FastAPI, SQLAlchemy, AsyncIO
- **Frontend:** React 19, TypeScript, Vite, Ant Design
- **Database:** PostgreSQL 16
- **Infrastructure:** Docker, Docker Compose, Nginx
