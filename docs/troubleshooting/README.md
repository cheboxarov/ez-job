# Troubleshooting

Руководство по решению проблем в проекте "AutoOffer".

## Как использовать этот раздел

1. Определите категорию проблемы
2. Найдите соответствующий раздел
3. Следуйте инструкциям по решению
4. Если проблема не решена, проверьте логи и свяжитесь с поддержкой

## Категории проблем

- **[Частые проблемы](common-issues.md)** — типичные проблемы при установке и запуске
- **[Проблемы с БД](database-issues.md)** — проблемы с базой данных и миграциями
- **[Проблемы с API](api-issues.md)** — ошибки API и интеграций
- **[Проблемы при деплое](deployment-issues.md)** — проблемы при развертывании

## Сбор информации для диагностики

Перед обращением за помощью соберите следующую информацию:

1. **Версия системы** — версия backend и frontend
2. **Логи** — логи приложения и ошибки
3. **Окружение** — ОС, версии Python, Node.js
4. **Шаги воспроизведения** — что вы делали перед ошибкой
5. **Ожидаемое поведение** — что должно было произойти

## Логи

Логи приложения находятся в:
- Backend: `backend/logs/`
- Docker: `docker logs <container_name>`
