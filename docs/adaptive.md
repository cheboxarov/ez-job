# Анализ адаптивности страниц

Документ содержит список страниц, которые не имеют адаптивности или будут плохо смотреться на мобильных устройствах.

## Критические проблемы (требуют срочного исправления)

### 1. AdminUsersPage (`frontend/src/pages/admin/AdminUsersPage.tsx`)
**Проблемы:**
- Таблица `Table` без адаптивности - на мобильных будет горизонтальная прокрутка
- Поле поиска `Input` с фиксированной шириной `300px` - не адаптируется на маленьких экранах
- Нет адаптивных классов для `Col` компонентов
- Фильтры в `Space` без адаптивности

**Рекомендации:**
- Добавить `scroll={{ x: true }}` для таблицы или использовать адаптивный режим
- Заменить фиксированную ширину на `width: '100%'` с `maxWidth`
- Использовать адаптивные `Col` с `xs`, `sm`, `md` пропсами

### 2. AdminMetricsDashboardPage (`frontend/src/pages/admin/AdminMetricsDashboardPage.tsx`)
**Проблемы:**
- Использование `Col span={6}` и `Col span={8}` без адаптивных пропсов - на мобильных все карточки будут в одну колонку, но слишком узкие
- Фильтры в `Space wrap` могут перекрываться на маленьких экранах
- `Select` с фиксированной шириной `200px` и `150px`
- Графики `ResponsiveContainer` могут быть слишком маленькими на мобильных

**Рекомендации:**
- Заменить `span={6}` на `xs={24} sm={12} md={6}` для статистических карточек
- Заменить `span={8}` на `xs={24} sm={12} md={8}` для метрик откликов
- Сделать фильтры адаптивными с использованием `Col` или `flex-wrap`
- Использовать `width: '100%'` для `Select` компонентов

### 3. AdminPlansPage (`frontend/src/pages/admin/AdminPlansPage.tsx`)
**Проблемы:**
- Таблица `Table` без адаптивности - на мобильных будет горизонтальная прокрутка
- Много колонок в таблице (Название, Лимит, Период, Длительность, Цена, Действия) - не поместятся на мобильных

**Рекомендации:**
- Добавить `scroll={{ x: true }}` для таблицы
- Рассмотреть возможность скрытия некоторых колонок на мобильных через `responsive` проп
- Или использовать карточки вместо таблицы на мобильных устройствах

### 4. AdminUserDetailPage (`frontend/src/pages/admin/AdminUserDetailPage.tsx`)
**Проблемы:**
- `Descriptions` с `column={2}` - на мобильных будет слишком узко, лучше `column={1}`
- `Select` с фиксированной шириной `200px` - не адаптируется
- Карточки управления без адаптивности

**Рекомендации:**
- Использовать `responsive` проп для `Descriptions` или условно `column={window.innerWidth < 768 ? 1 : 2}`
- Заменить фиксированную ширину `Select` на `width: '100%'`
- Добавить адаптивные отступы для карточек

### 5. AdminLlmCallsPage (`frontend/src/pages/admin/AdminLlmCallsPage.tsx`)
**Проблемы:**
- Таблица `Table` без адаптивности - очень много колонок (Дата, Агент, Модель, Пользователь, Статус, Токены, Стоимость, Длительность, Действия)
- Фильтры с фиксированными ширинами `200px` и `150px`
- `Drawer` с `width="90%"` может быть слишком широким на мобильных
- `Input` с фиксированной шириной `200px`

**Рекомендации:**
- Добавить `scroll={{ x: true }}` для таблицы
- Использовать адаптивные фильтры с `Col` компонентами
- Изменить `Drawer` на `width={window.innerWidth < 768 ? '100%' : '90%'}`
- Заменить фиксированные ширины на адаптивные

## Средние проблемы (желательно исправить)

### 6. ChatDetailPage (`frontend/src/pages/ChatDetailPage.tsx`)
**Проблемы:**
- Сообщения с `maxWidth: '75%'` - на мобильных может быть слишком узко или широко
- `maxHeight: 'calc(100vh - 280px)'` может быть проблемой на маленьких экранах
- Кнопка отправки с фиксированной шириной `44px` - нормально, но можно улучшить

**Рекомендации:**
- Использовать `maxWidth: window.innerWidth < 768 ? '90%' : '75%'` для сообщений
- Добавить медиа-запросы для высоты контейнера сообщений
- Рассмотреть использование `useWindowSize` хука для адаптивности

**Рекомендации:**
- Добавить `padding: '0 16px'` для мобильных версий
- Использовать адаптивные `Col` с `xs={24} md={12}` для кнопок

## Приоритеты исправления

1. **Высокий приоритет:** AdminUsersPage, AdminMetricsDashboardPage, AdminPlansPage, AdminUserDetailPage, AdminLlmCallsPage

## Общие рекомендации

1. Всегда использовать адаптивные пропсы для `Col`: `xs`, `sm`, `md`, `lg`, `xl`
2. Избегать фиксированных ширин в пикселях, использовать `width: '100%'` с `maxWidth`
3. Для таблиц добавлять `scroll={{ x: true }}` или использовать адаптивный режим
4. Использовать медиа-запросы или хуки для условного рендеринга на мобильных
5. Тестировать на реальных мобильных устройствах или эмуляторах
