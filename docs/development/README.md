## Процесс разработки

1. Создайте ветку для новой фичи
2. Разработайте фичу согласно архитектуре (Domain → Infrastructure → Application → Presentation)
3. Напишите тесты
4. Создайте Pull Request
5. Пройдите Code Review
6. После одобрения — мерж в main

Подробнее см. [Добавление новых фич](adding-features.md).

## Стандарты

- **Python:** PEP 8, типизация, docstrings
- **TypeScript:** ESLint правила, форматирование
- **Git:** Conventional Commits

Подробнее см. [Стиль кода](code-style.md).
