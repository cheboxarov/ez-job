# Эндпоинты: Чаты

Документация эндпоинтов для работы с чатами HeadHunter.

## Базовый путь

```
/api/chats
```

## Получение списка чатов

**Endpoint:** `GET /api/chats`

**Описание:** Получение списка чатов пользователя с работодателями.

**Аутентификация:** Требуется

**Query Parameters:**
- `unread_only` (bool, optional) — только непрочитанные

**Response:** `200 OK`
```json
{
  "chats": [
    {
      "id": "12345678",
      "vacancy_id": "87654321",
      "employer_name": "Company Name",
      "last_message": {...},
      "unread_count": 2
    }
  ]
}
```

## Получение деталей чата

**Endpoint:** `GET /api/chats/{chat_id}`

**Описание:** Получение детальной информации о чате и сообщениях.

**Аутентификация:** Требуется

**Response:** `200 OK` — детали чата с сообщениями

## Отправка сообщения

**Endpoint:** `POST /api/chats/{chat_id}/messages`

**Описание:** Отправка сообщения в чат.

**Аутентификация:** Требуется

**Request Body:**
```json
{
  "text": "Текст сообщения"
}
```

**Response:** `201 Created`

## Отметка сообщения как прочитанного

**Endpoint:** `POST /api/chats/{chat_id}/mark-read`

**Описание:** Отметка сообщений в чате как прочитанных.

**Аутентификация:** Требуется

**Response:** `200 OK`
